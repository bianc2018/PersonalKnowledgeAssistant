## Summary

为项目新增一个单一 Python CLI 部署脚本 `deploy.py`，实现从代码克隆到服务运行的"一键部署"。脚本自动完成环境检查、依赖安装、配置模板生成、数据库初始化（委托给应用启动流程）、端口冲突处理和前台服务启动。全程在终端输出清晰的步骤进度，并对错误场景（端口占用、网络中断、进程冲突）提供明确的反馈和处理策略。

## Specification

- **FR-001**: 提供单一 Python CLI 命令 `python deploy.py`，自动化构建、部署和运行工作流
- **FR-002**: 验证目标环境具备 Python 3.11+、pip 及端口检测所需工具
- **FR-003**: 首次部署时由 FastAPI 应用在启动时自动执行 SQLAlchemy `create_all()` 初始化数据库
- **FR-004**: 服务前台运行，日志直接输出到终端，用户可通过 Ctrl+C 停止
- **FR-005**: 部署失败时显示清晰的错误提示，包含具体失败的步骤
- **FR-006**: 支持在现有环境上重复部署且不破坏已有数据
- **FR-007**: 缺失必需配置文件时自动生成模板并暂停，提示用户补全后重新运行
- **FR-008**: 实时输出每个部署步骤的执行状态和进度
- **FR-009**: 检测端口冲突并动态分配可用替代端口，同时向用户明确告知新端口
- **FR-010**: 检测到应用服务已在运行时提示用户手动停止现有进程，而非自动重启或启动多实例
- **FR-011**: 网络中断导致依赖下载失败时最多重试 3 次，全部失败后退出并报告错误

完整需求与验收标准详见 [spec.md](../spec.md)。

## Implementation

- **技术架构**: 部署脚本采用纯 Python 标准库实现，无额外 CLI 框架或系统守护进程依赖
- **数据库初始化**: 委托给既有 FastAPI 启动流程，避免重复实现迁移脚本
- **项目结构**: 采用单项目结构，在根目录新增 `deploy.py` 作为部署入口，不改动原有 `src/` 和 `tests/` 模块划分
- **核心实现**:
  - 创建 `deploy.py` 骨架和 `DeploymentConfig` 数据类
  - 实现首次一键启动服务的环境检查、依赖安装、端口分配和服务启动
  - 实现更新后重新部署的运行中服务检测（扫描 `/proc/*/cmdline`）
  - 实现部署失败时的清晰反馈与错误分类建议
  - 补充单元测试和端到端测试

详细实现计划与架构决策见 [plan.md](../plan.md) 和 [tasks.md](../tasks.md)。
