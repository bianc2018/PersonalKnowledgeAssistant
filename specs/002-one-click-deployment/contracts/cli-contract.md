# CLI 契约：一键部署脚本

> 命令入口: `python deploy.py`

---

## 1. 命令概述

`deploy.py` 是一个纯 Python 标准库实现的 CLI 脚本，位于项目根目录。它负责自动完成从环境检查到服务启动的全流程部署。

---

## 2. 执行步骤与输出契约

### 2.1 标准成功流程

```text
[1/5] 检查环境 ........................ [OK]
[2/5] 安装依赖 ........................ [OK]
[3/5] 检查配置 ........................ [OK]
[4/5] 检查端口与进程冲突 ............... [OK]
[5/5] 启动服务 ........................ [OK]

服务已启动，访问地址: http://127.0.0.1:8000
按 Ctrl+C 停止服务
```

**字段说明**:
- 每行格式固定为 `[N/5] 步骤名称 ........................ [STATUS]`
- `STATUS` 取值: `[OK]` / `[FAIL]` / `[SKIP]`
- 若端口被动态分配，最终输出中的 URL 应显示实际端口

---

### 2.2 环境检查失败

```text
[1/5] 检查环境 ........................ [FAIL]

错误: Python 版本不符合要求
当前版本: 3.10.12
所需版本: >= 3.11
建议: 请安装 Python 3.11 或更高版本后重试
```

**错误输出要求**:
- 必须包含 `错误:` 前缀
- 必须指出具体失败步骤
- 必须提供可操作的修复建议

---

### 2.3 配置模板生成

当 `.env` 文件缺失时：

```text
[3/5] 检查配置 ........................ [FAIL]

错误: 未找到环境配置文件 (.env)
已自动生成模板文件: .env
请打开该文件，填写必要的配置项后再次运行 python deploy.py
```

**生成的 `.env` 模板内容**:

```bash
# AI 知识管理助手 - 环境配置模板
# 请根据实际情况填写以下字段

# 数据库路径（可选，默认使用项目根目录下的 data/app.db）
# DATABASE_URL=./data/app.db

# 会话密钥（生产环境必须修改）
SECRET_KEY=change-me-in-production

# 文件上传存储目录
# FILES_DIR=./files

# 日志存储目录
# LOG_DIR=./logs

# LLM 接口配置
LLM_BASE_URL=
LLM_API_KEY=
LLM_MODEL=
# LLM_ENABLE_SEARCH=false

# Embedding 接口配置
EMBEDDING_BASE_URL=
EMBEDDING_API_KEY=
EMBEDDING_MODEL=

# 搜索 API 配置（可选）
# SEARCH_PROVIDER=
# SEARCH_API_KEY=
# SEARCH_BASE_URL=
```

### 2.4 进程冲突检测

当检测到服务已在运行时：

```text
[4/5] 检查端口与进程冲突 ............... [FAIL]

错误: 检测到应用服务已在运行
端口 8000 当前被占用。
请先停止现有进程，然后再次运行 python deploy.py
提示: 在终端中按下 Ctrl+C 即可停止前台运行的服务
```

### 2.5 依赖安装网络失败

```text
[2/5] 安装依赖 ........................ [FAIL]

错误: 依赖安装失败，已重试 3 次
最后一条错误信息: 
  Connection timed out
建议: 请检查网络连接，或稍后重试。如果使用了代理，请确保代理配置正确。
```

## 3. Uvicorn 启动参数契约

部署脚本成功通过所有检查后，必须以以下参数启动服务：

```bash
{python_executable} -m uvicorn src.main:app --host 127.0.0.1 --port {actual_port}
```

**参数约束**:
- `--host` 固定为 `127.0.0.1`
- `--port` 为动态探测后的实际可用端口
- 不使用 `--reload`（避免生产环境误用，开发时手动添加）
- 前台运行，子进程 stdout/stderr 直接绑定到当前终端

---

## 4. 退出码契约

| 退出码 | 含义 | 场景 |
|--------|------|------|
| `0` | 成功 | 服务正常启动并运行中（用户手动停止后亦为 0） |
| `1` | 通用失败 | 任意部署步骤失败（环境检查、依赖安装、配置缺失、进程冲突等） |
| `130` | 用户中断 | 服务运行期间用户按下 Ctrl+C（继承自 uvicorn 子进程 SIGINT） |

---

## 5. 文件系统副作用契约

`deploy.py` 执行过程中可能创建或修改的文件：

| 路径 | 操作 | 条件 |
|------|------|------|
| `.env` | 生成模板 | 文件不存在时 |
| `data/` | 创建目录 | 目录不存在时 |
| `logs/` | 创建目录 | 目录不存在时 |
| `.venv/` | 不修改 | 由用户自行管理（若存在则使用当前解释器） |

**约束**:
- 脚本不得修改 `src/` 目录中的任何源代码文件
- 脚本不得删除已有的数据库或用户上传文件
