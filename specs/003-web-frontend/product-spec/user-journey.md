## Journey Map

| Step | User Action | System Response | Emotion | Notes |
|------|-------------|-----------------|---------|-------|
| 1 | 打开浏览器访问 `http://localhost:8000` | 若未初始化，显示密码设置页；若已初始化，显示登录页 | 🙂 | 首次使用的引导至关重要 |
| 2 | 设置/输入密码并点击登录 | 校验成功后返回 JWT，前端存储 token 并跳转 Dashboard | 🙂 | 密码规则实时校验 |
| 3 | 进入 Dashboard | 展示知识总数、系统状态卡片、最近会话、快捷入口 | 🙂 | 空状态时应显示明显的 CTA |
| 4 | 点击"添加知识"进入知识库页 | 展示知识列表、搜索框、标签筛选器 | 🙂 | 首次使用列表为空，提示"创建第一条知识" |
| 5 | 选择"上传文件"并提交 PDF | 前端上传中显示进度；上传成功后列表刷新，展示新条目 | 🙂 | 文件过大或格式不支持时 inline 提示 |
| 6 | 点击知识条目进入详情 | 展示内容、标签、附件、置信度、版本历史 | 🤔 | 置信度和版本是差异化信息 |
| 7 | 点击"开始对话"进入 Chat | 左侧会话列表，右侧新开会话，底部输入框 | 🙂 | 若已有相关会话也可选择继续 |
| 8 | 输入问题并发送 | 前端调用 SSE API，消息区域逐字流式输出回答；带引用 chip | 😮 | Typing indicator 和流式输出建立临场感 |
| 9 | 悬停回答中的引用 | 弹出该来源知识的标题和摘要 | 🙂 | 建立信任，避免黑盒 |
| 10 | 返回 Dashboard，点击"新建调研" | 进入 Research 页面，输入主题和范围描述后提交 | 🤔 | 主题宽泛时可能需要后续细化 |
| 11 | 进入调研任务详情 | 前端通过 fetch stream 消费 SSE，展示进度条和阶段性摘要 | 🙂 | "正在检索…" → "正在分析…" |
| 12 | 系统抛出决策问题 | 页面暂停并展示选择/输入表单，等待用户决策 | 😐 | `awaiting_input` 状态，无超时 |
| 13 | 用户提交决策 | 前端调用 respond API，SSE 继续推送，最终展示完整报告 | 🙂 | 报告以 Markdown 渲染 |
| 14 | 点击"保存到知识库" | 保存成功后 Toast 提示，可跳转至对应知识详情 | 🙂 | 报告变为可搜索、可引用 |
| 15 | 进入 Settings | 配置 LLM API Key、调整隐私开关、执行导出/导入 | 🙂 | 危险操作（重置）需二次密码确认 |

## Alternative Paths

### Path B: 直接通过 URL 访问深层页面
- 用户输入 `http://localhost:8000/#/chat/conv-123`。
- 若 token 有效：前端加载对应会话和消息。
- 若 token 无效：拦截 401，保存目标路径，跳转登录页，登录后自动跳回目标页。

### Path C: 服务重启后继续使用
- 用户发送任意 API 请求。
- 后端返回 401（master key 缓存丢失）。
- 前端全局拦截器捕获 401，清除 token，展示"服务端已重启，请重新登录"，跳转 `#/login`。

### Path D: LLM 不可用时的降级体验
- Dashboard 展示黄色警告 Banner："LLM 服务不可用，对话和调研功能受限"。
- Chat 页面仍可输入，但后端返回降级文案，前端正常展示在消息流中。
- Research 页面仍可提交任务；若配置了本地模型，任务降级执行并在结果中标注"本地模式"。

## Error Scenarios

| Scenario | System Response | Recovery |
|----------|-----------------|----------|
| 密码错误 | 登录页 inline error："密码错误" | 用户重试 |
| 网络断开 | Toast："网络不可用，请检查连接" | 自动重试或手动刷新 |
| SSE 连接中断 | Chat/Research 页面提示"连接已断开，正在重试…" | 前端自动重连（最多 3 次） |
| 导入 ZIP 损坏 | 设置页展示后端返回的详细错误信息 | 用户更换文件重试 |
| 上传文件 >1GB | 前端即时拦截："单文件不得超过 1GB" | 用户更换文件 |
| 内容长度 <5 字符 | 表单 inline error："内容至少需要 5 个字符" | 用户补充内容 |

## Journey Metrics

- **Entry point:** 浏览器直接访问服务根路径 `/`
- **Exit point:** 用户完成一次有意义操作（添加知识 / 获得流式回答 / 保存调研报告）
- **Expected completion time (首次使用):** 从打开浏览器到发送第一条 Chat 消息 ≤3 分钟
- **Drop-off risk points:**
  1. 首次使用时的密码设置页（复杂度要求）
  2. 文件上传失败且错误提示不明确
  3. SSE 流式输出异常导致 Chat 体验中断
