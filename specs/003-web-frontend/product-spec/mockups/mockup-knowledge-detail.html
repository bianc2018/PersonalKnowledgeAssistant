FastAPI Streaming Responses

backend python fastapi sse

Overview

FastAPI supports multiple response types for streaming data: plain text iterables, StreamingResponse, and server-sent events (SSE). Choosing the right one depends on your use case.

SSE with StreamingResponse

Use EventSourceResponse or a custom generator with text/event-stream media type to push real-time updates to the client without polling.

  • Lower overhead than WebSockets for one-way updates.
  • Native retry and event-id handling.
  • Works well with HTTP proxies and load balancers.

Trade-offs

Streaming is great for large payloads and real-time AI responses, but error handling mid-stream can be tricky. Always validate headers and handle client disconnects gracefully.
