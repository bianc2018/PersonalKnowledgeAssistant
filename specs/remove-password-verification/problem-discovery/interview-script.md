# User Interview Script: 去除密码校验（个人本地部署模式）

> Goal: 验证"个人本地部署场景下，强制密码校验构成了不必要的使用 friction"这一问题是否真实、普遍且值得优先解决。
> Duration: 20–30 minutes | Format: 1:1 半结构化访谈
> Recruit: 在个人电脑、NAS 或家庭服务器上部署使用过本地知识管理/笔记/AI 工具的个人用户

---

## Warm-up（5 min）

1. 请简单介绍一下你平时是怎么使用这个 AI 知识管理助手的？（部署在哪、使用频率）
2. 你通常是唯一使用它的人，还是也会分享给家人/朋友/同事？
3. 你平时打开它的第一步是什么？

---

## Problem Exploration（10 min）

4. 描述一下你最近一次打开这个工具的经历。过程中有没有让你觉得"又来了"的繁琐步骤？
5. 关于登录/密码这件事，它对你来说意味着什么？是一个必要的安全措施，还是一个多余的步骤？
6. 你有没有忘记过这个工具的密码？当时你是怎么解决的？
7. 如果这个世界上没有这个密码墙，你的使用习惯会有什么变化？
8. 用 1–10 分打分，"每次都需要登录"这件事对你使用该工具的意愿有多大影响？

---

## Current Solution Probe（8 min）

9. 你有没有用过其他类似需要在本地部署的工具（比如 Jupyter Notebook、Ollama、Obsidian Sync 等）？它们是怎么处理访问控制的？
10. 相比之下，你更希望这个工具怎么做？
    - A. 保持现状：每次都需要密码
    - B. 可选：启动时可以设置"不需要密码"
    - C. 完全无密码：本地访问永远不需要密码
    - D. 其他（请描述）
11. 如果提供"忘记密码后一键重置"的功能，但代价是清空本地数据，你能接受吗？为什么？

---

## Validation Probe（5 min）

12. 想象一下，下次更新后，启动脚本出现了一个选项：
    - `[1] 启用密码保护（当前行为）`
    - `[2] 关闭密码保护（仅限本地单人使用）`
    - `[3] 重置密码并清空数据`
    你会怎么选？这个选项是否符合你的心理预期？
13. 如果把"关闭密码保护"推荐给同样在个人电脑上部署工具的朋友，你会不会有什么顾虑？

---

## Wrap-up（2 min）

14. 还有什么我没问到、但你觉得和这个功能相关的使用体验想补充的吗？

---

## Scoring Rubric（每场访谈后填写）

| 信号 | 1–5 分 | 备注 |
|------|--------|------|
| 问题频率（每次使用都遇到吗？） | | |
| 当前痛苦程度（是否需要 workaround？） | | |
| 对"可选密码"方案的接受度 | | |
| 对"重置密码=清空数据"的接受度 | | |
| **总分** | /20 | |

> **评分标准：**
> - ≥ 15/20 → 强信号，问题真实且值得解决
> - 10–14/20 → 中等信号，可能受特定用户群体限制，建议补充访谈
> - < 10/20 → 弱信号，建议重新审视问题定义或优先级
